github制作个人博客简单步骤

发表于 2017-12-29

软件安装

  • 安装github 下载地址
  • 安装node.js、npm (Node.js是一个Javascript运行环境,npm是Nodejs的包管理器)

Github配置

  1. 创建github账号；
  2. 创建github仓库，以github用户名命名。若github用户名是test，则新建一个test.github.io的仓库（必须是你的github用户名，其它名称无效），网站访问地址即http://test.github.io ；
  3. 配置SSH key，将本地仓库与github仓库连接；
  4. 可选择注册新域名并绑定，新建CNAME文件，内容是新域名，在github设置中配置GitHub Pages的Custom domain；

Hexo命令

Hexo是一个简单、快速、强大的基于 Github Pages 的博客发布工具，支持Markdown格式，有众多优秀插件和主题。

  • 安装命令 $ npm install -g hexo；
运行hexo需要运行cd切换到对应目录下：
  • 初始化命令 $ hexo init；
  • 新建文章 $ hexo new "postName"；
  • 新建页面 $ hexo new page "pageName"；
  • 生成命令 $ hexo g；
  • 启动服务命令 $ hexo s，启动服务后可在本地端口预览，在浏览器输入http://localhost:4000；
  • 部署命令 $ hexo d，需在系统配置文件内配置deploy的部分；

注意事项：hexo有2种_config.yml配置文件，一个是根目录下的全局_config.yml，一个是各个theme下的_config.yml。

易错点：修改配置文件过程中谨记，在冒号之后必有空格！！！

快捷键：Control+p，可以查找之前输入的命令

主题下载

下载命令为 $ git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia，该主题所在github仓库地址+theme+主题名称。修改全局配置文件，找到theme关键词，修改主题，重新生成部署；

建议先执行$ hexo clean，清理public的内容，然后再重新生成和发布，可避免一些问题；

易错点： 修改配置文件过程中谨记，在冒号之后必有空格！！！

Hexo简单书写

  • 下载markdown编辑器，编辑在/source/_post/下面新生成的“新建.m”文件；

  • 标题样式：# 一级标题## 二级标题### 三级标题#### 四级标题##### 五级标题；

  • 列表样式：星号* + 空格；数字 + 空格

  • 引用样式：在需要引用的文本前面添加>

  • 图片样式：感叹号！ + 方括号[] + 圆括号()【方括号内输入图片名称，圆括号内输入图片地址，可是本地图片路径，或者图片网址】；

  • 链接样式：方括号[] + 圆括号()【方括号内输入链接文字，圆括号内输入网站】；

  • 代码框样式：用一个 ` 把中间的代码包含起来；
